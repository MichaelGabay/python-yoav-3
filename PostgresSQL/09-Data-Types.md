# טיפוסי נתונים

## טיפוסים מספריים

| טיפוס | תיאור | דוגמה |
|-------|--------|-------|
| `INTEGER` / `INT` | מספר שלם | 42, -7 |
| `BIGINT` | שלם גדול | 9007199254740991 |
| `SMALLINT` | שלם קטן | -32768 עד 32767 |
| `DECIMAL(p,s)` | עשרוני מדויק | DECIMAL(10,2) → 1234.56 |
| `NUMERIC(p,s)` | כמו DECIMAL | |
| `REAL` | נקודה צפה | 3.14 |
| `DOUBLE PRECISION` | דיוק כפול | 3.14159265359 |

## מחרוזות

| טיפוס | תיאור |
|-------|--------|
| `VARCHAR(n)` | מחרוזת באורך משתנה, מקסימום n תווים |
| `CHAR(n)` | מחרוזת באורך קבוע n |
| `TEXT` | מחרוזת באורך לא מוגבל |

## תאריכים וזמנים

| טיפוס | תיאור | דוגמה |
|-------|--------|-------|
| `DATE` | תאריך | '2025-02-11' |
| `TIME` | שעה | '14:30:00' |
| `TIMESTAMP` | תאריך + שעה | '2025-02-11 14:30:00' |
| `INTERVAL` | טווח זמן | INTERVAL '1 day' |

## בוליאני

- `BOOLEAN` – `TRUE`, `FALSE`, `NULL`

## מיוחדים

| טיפוס | תיאור |
|-------|--------|
| `SERIAL` | INTEGER אוטומטי (לא null, ייחודי) |
| `UUID` | מזהים ייחודיים גלובליים |
| `JSON` / `JSONB` | נתונים מובנים |

## תרגילים

### תרגיל 1
בחר טיפוס מתאים לכל שדה: מחיר מוצר, מספר טלפון (ספרות), שם מלא, תאריך לידה.

### תרגיל 2
מה ההבדל בין `VARCHAR(10)` ל-`CHAR(10)`? מתי תשתמש בכל אחד?

### תרגיל 3
צור עמודה `price` במחירים עד 999,999.99 עם שני מקומות אחרי הנקודה. איזה טיפוס מתאים?

### תרגיל 4
מה ההבדל בין `TIMESTAMP` ל-`TIMESTAMPTZ`? מתי נשתמש ב-TIMESTAMPTZ?

### תרגיל 5
הטבלה `logs` צריכה: `id` אוטומטי, `message` (טקסט ארוך), `level` ('INFO','WARN','ERROR'), `created_at`. כתוב את ה-CREATE TABLE המלא עם טיפוסים מתאימים.

## פתרונות

<details>
<summary>לחץ כדי לראות פתרונות</summary>

### תרגיל 1
- מחיר: `DECIMAL(10,2)` או `NUMERIC(10,2)`  
- טלפון: `VARCHAR(15)` (כולל סימנים) או `BIGINT` (רק ספרות)  
- שם מלא: `VARCHAR(100)` או `TEXT`  
- תאריך לידה: `DATE`  

### תרגיל 2
`VARCHAR(10)` – אורך משתנה עד 10 (הטקסט "hi" תופס 2). `CHAR(10)` – תמיד 10 תווים (משלים רווחים). משתמשים ב-CHAR כשהאורך קבוע (למשל קוד מדינה).

### תרגיל 3
`DECIMAL(8,2)` – 6 ספרות לפני הנקודה, 2 אחרי.

### תרגיל 4
`TIMESTAMP` – ללא אזור זמן. `TIMESTAMPTZ` – עם אזור זמן, נשמר ב-UTC. להשתמש ב-TIMESTAMPTZ כשעובדים עם משתמשים באזורי זמן שונים.

### תרגיל 5
```sql
CREATE TABLE logs (
    id         SERIAL PRIMARY KEY,
    message    TEXT,
    level      VARCHAR(10) CHECK (level IN ('INFO', 'WARN', 'ERROR')),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
